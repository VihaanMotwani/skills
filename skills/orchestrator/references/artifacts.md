# Workflow Artifacts

Use these project artifacts for greenfield apps:

- `CONTEXT.md`: glossary, domain language, and product assumptions only
- `docs/build/specs/YYYY-MM-DD-<topic>-spec.md`: reviewed product spec
- `docs/adr/YYYY-MM-DD-<decision>.md`: meaningful hard-to-reverse decisions
- `docs/build/slices.md`: lightweight slice ladder
- `docs/build/current-slice.md`: detailed next-slice contract only
- `docs/build/truth-ledger.md`: real / stubbed / not built / broken or unknown
- `docs/build/handoffs/YYYY-MM-DD-<phase>-slice-<n>.md`: session handoff

Do not create all artifacts up front. Create them when the phase needs them.
